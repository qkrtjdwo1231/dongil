# 동일유리 작업관리 PRD

## 1. 프로젝트 개요

### 1.1 서비스 목적
- 동일유리의 주문/업로드/분석 업무를 웹에서 통합 관리한다.
- 업로드한 생산실적 파일과 저장된 데이터를 기반으로 팀장/대표가 조회, 관리, 분석, AI 질의를 수행할 수 있게 한다.
- 기존 직원 등록 기능은 유지하되, 현재 핵심 운영 화면은 팀장/대표 콘솔로 확장한다.

### 1.2 주요 역할
- 직원: 주문 등록과 업로드 보조
- 팀장: 프로젝트, 일정, 팀 관리, 현장 운영 확인
- 대표: 생산실적 분석, 다차원 분석, 목표 관리, 데이터 그리드, AI 분석

## 2. 기능 요구사항

### 2.1 공통
- 상단 헤더에 역할 전환 UI 제공
- Supabase 환경변수 미설정 시 안전한 안내 메시지 표시
- 역할별로 서로 다른 화면 구조 제공

### 2.2 직원 화면
- 기본 등록
- 빠른 등록
- 주문 목록
- 체크리스트
- 즐겨찾기
- 파일 업로드

### 2.3 팀장 화면
- 좌측 메뉴 기반 콘솔 구조
- 메뉴:
  - 대시보드
  - 프로젝트
  - 일정
  - 팀 관리
  - 설정
- 검색 기준:
  - PID
  - 거래처
  - 현장
  - 품명
  - 등록자
- 현황 카드, 라인별 작업량, 우선 확인 목록, 프로젝트 집계, 등록자 집계 제공

### 2.4 대표 화면
- 좌측 메뉴 기반 분석 콘솔 구조
- 메뉴:
  - 대시보드
  - 다차원 분석
  - 목표 설정
  - 데이터 그리드
  - 데이터 임포트
  - AI 분석
  - 설정
- 업로드 기반 KPI 분석 제공
- 기간 필터, 집계 단위, 분석 기준, 측정 지표 변경 가능
- 월별 목표 등록/수정/삭제 가능
- 원본성 높은 데이터 테이블 조회 가능
- 업로드 파일 기반 AI 분석 가능

### 2.5 데이터 임포트
- `.xlsx`, `.xls`, `.csv` 업로드 지원
- 미리보기, 유효 행 수, 검토 필요 행 수 표시
- 원본 파일 Storage 저장
- 업로드 행 데이터 `uploaded_rows`에 저장
- `pid`와 `raw_payload`를 함께 보존

### 2.6 AI 분석
- 업로드한 파일과 저장된 행 데이터만 기준으로 답변
- 파일에 없는 내용은 추측하지 않음
- AI 기억 규칙(`ai_memory_rules`) 관리 가능
- 업로드 기반 추천/실시간 후보 제공

## 3. 비기능 요구사항

### 3.1 성능
- 대용량 파일도 원본 전체를 AI에 직접 전달하지 않음
- 저장된 행 데이터에서 관련 데이터만 검색 후 AI에 전달

### 3.2 유지보수성
- 역할별 화면 분리
- 업로드/AI/분석/등록 기능을 컴포넌트와 유틸 단위로 분리

### 3.3 보안
- Gemini API 키는 서버 환경변수로만 관리
- 업로드 파일 접근은 API 라우트를 통해 제어

## 6. UI/UX 요구사항
- 현재 앱의 밝고 정돈된 업무용 SaaS 스타일 유지
- 디자인보다 기능 구조를 우선
- 팀장/대표 화면은 조회/관리/분석 중심
- 직원 화면은 입력 중심
- 팀장/대표는 상단 검색 + 좌측 메뉴 + 본문 카드/표 구조 사용
