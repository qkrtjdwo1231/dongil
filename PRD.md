# 동일유리 작업관리 PRD

## 1. 프로젝트 개요

### 1.1 서비스 목적
- 동일유리의 MES 포장공정 실적 파일을 업로드하고, 대표와 팀장이 생산 부담과 라인 부하를 빠르게 파악할 수 있는 내부용 분석 웹을 제공한다.
- 이 서비스는 단순 엑셀 조회기가 아니라, 대용량 생산실적 로그를 구조화하고 핵심 경영/운영 인사이트를 자동으로 도출하는 AI 분석 대시보드다.
- 분석 결과는 항상 `이 데이터만으로 가능한 해석`과 `추가 데이터가 있어야 가능한 해석`을 구분해서 보여준다.

### 1.2 핵심 사용자
- 대표: 생산 부담, 제품 믹스, 거래처 의존도, 라인 편중, 데이터 품질, 이상징후, 액션 아이템 확인
- 팀장: 라인별 운영 현황, 프로젝트/일정/팀 관리, 업로드 데이터 검토, 예외 상황 확인

### 1.3 핵심 한 줄
- 엑셀 기반 포장공정 데이터를 업로드하면, 라인·품목·거래처·현장별 생산 부담을 자동 분석하고 AI가 경영 인사이트를 요약해주는 웹앱

### 1.4 분석 데이터 범위
- 이 서비스는 매출/원가/마진 분석 시스템이 아니다.
- 핵심 분석 범위는 다음과 같다.
  - 생산 부담 분석
  - 포장공정 실적 분석
  - 라인 부하 분석
  - 제품 믹스 분석
  - 거래처 의존도 분석
  - 데이터 품질 분석
- 따라서 수익성, 마진, 영업이익을 단정하는 표현은 사용하지 않는다.

## 2. 업로드 데이터 정의

### 2.1 기준 컬럼

- PID
- 공정
- 품목코드
- 품명
- 가로
- 세로
- 수량
- 평수
- 의뢰번호
- NO
- 거래처

- S
- 라인
- 등록자

### 2.2 데이터 특성 해석
- 데이터는 주문서가 아니라 포장공정 실적 로그다.
- 1행은 포장공정에서 실제 처리된 실적 단위로 본다.
- 대표 화면의 목적은 입력 보조가 아니라, 실적 해석과 이상징후 탐지다.

## 3. 기능 요구사항

### 3.1 엑셀 업로드
- 사용자는 `.xlsx`, `.xls`, `.csv` 파일을 업로드할 수 있다.
- 업로드 시 원본 파일은 Storage에 저장한다.
- 파싱된 행 데이터는 DB에 저장한다.
- 업로드 직후 구조화/검증/기본 집계를 수행한다.
- 등록일시는 날짜, 월, 시간대 분석에 활용할 수 있도록 구조화한다.

### 3.2 데이터 정제
- 평수, 가로, 세로, 수량을 숫자형으로 변환한다.
- 거래처, 품명, 현장 값의 앞뒤 공백을 제거한다.
- 현장명이 비어 있으면 `현장 미입력`으로 정규화한다.
- 라인 값은 `1-LINE`, `2-LINE` 기준으로 정규화한다.
- PID 중복 여부를 체크한다.
- 가로/세로/평수의 0 또는 음수 값을 검토 대상으로 표시한다.

### 3.3 필수 컬럼 검증
- 다음 컬럼의 존재 여부를 업로드 시 검증한다.
  - 등록일시
  - PID
  - 공정
  - 품명
  - 수량
  - 거래처
  - 라인
- 누락 시 업로드는 저장하되, 분석 신뢰도 경고를 함께 표시한다.

### 3.4 제품군 자동 분류
- 품명에 포함된 키워드를 기준으로 제품군을 자동 분류한다.
- 분류 규칙:
  - `접합`, `PVB`, `안전` 포함 → `접합/안전`
  - `로이`, `LOW-E`, `SKN`, `MCT`, `MZT` 포함 → `로이/코팅`
  - `Ar`, `아르곤`, `단열` 포함 → `단열/아르곤`
  - 그 외 → `일반/복층`
- 우선순위:
  1. 접합/안전
  2. 로이/코팅
  3. 단열/아르곤
  4. 일반/복층

### 3.5 업로드 구조화 결과
- 업로드 직후 아래 정보를 파일 단위 요약으로 생성한다.
  - 분석 기간
  - 총 행 수
  - 총 수량
  - 총 평수
  - 평균 평수
  - 거래처 수
  - 현장 수
  - 품목 수
  - 라인 수
  - 등록자 수
  - PID 수
  - PID 누락 건수
  - 현장 누락 건수
  - 품명 누락 건수
  - 라인 누락 건수
  - 규격 누락 건수
  - 평수 이상값 건수
  - 상위 거래처/품목/라인/공정/등록자 분포
  - 이상징후 후보
  - 대표 체크 포인트

### 3.6 대표용 요약 대시보드
- 상단 KPI 카드:
  - 총 포장 건수
  - 총 수량
  - 총 평수
  - 평균 평수
  - 거래처 수
  - 현장 수
  - 품목 수
  - 라인 수
  - 상위 5개 거래처 평수 비중
  - 상위 5개 품목 평수 비중
  - 현장 누락률
- 대표 대시보드는 숫자 나열이 아니라 `우선 검토`, `리스크`, `액션` 중심으로 보여준다.
- 대표가 가장 먼저 봐야 할 3~5개 항목을 자동 추출한다.

### 3.7 대표용 차트/시각화
- 월별 총 평수 추이
- 라인별 총 평수 비교
- 제품군별 평수 비중
- 거래처 TOP 10
- 품목 TOP 10
- 월별 제품군 변화
- 라인별 월별 평수 추이
- 시간대별 등록 건수 / 평수

### 3.8 라인 분석 페이지
- 분석 목적:
  - 1-LINE과 2-LINE이 얼마나 처리했는지뿐 아니라, 어떤 제품군을 처리했는지 비교한다.
- 필요 지표:
  - 라인별 총 평수
  - 라인별 건수
  - 라인별 평균 평수
  - 라인별 제품군 구성
  - 라인별 월별 평수 추이
  - 라인별 거래처 TOP 10
- 인사이트는 라인 편중, 제품군 편중, 품질 부담 가능성을 설명해야 한다.

### 3.9 품목/제품군 분석 페이지
- 분석 목적:
  - 주력 품목과 제품 믹스를 파악한다.
- 필요 지표:
  - 품목별 총 평수 TOP 20
  - 품목별 건수 TOP 20
  - 제품군별 평수 비중
  - 월별 제품군 변화
  - 품목별 평균 평수
  - 품목별 라인 편중도
- 대표가 원재료 준비와 생산계획 관점에서 볼 수 있도록 상위 품목 집중도를 표시한다.

### 3.10 거래처/현장 분석 페이지
- 분석 목적:
  - 어떤 거래처와 현장이 포장공정 부담을 크게 만드는지 파악한다.
- 필요 지표:
  - 거래처별 총 평수 TOP 20
  - 거래처별 건수 TOP 20
  - 현장별 총 평수 TOP 20
  - 거래처별 제품군 구성
  - 거래처별 평균 평수
  - 거래처별 현장 누락률
  - 상위 5개/10개 거래처 의존도
- 거래처 자동 분류:
  - 총 평수 상위권 → 핵심 물량 고객
  - 평균 평수 높음 → 대형 규격 고객
  - 로이/코팅 비중 높음 → 기능성 제품 고객
  - 현장 누락률 높음 → 데이터 관리 필요 고객
  - 특정 월만 물량 집중 → 프로젝트성 고객

### 3.11 시간대/작업 부하 분석 페이지
- 분석 목적:
  - 등록일시 기준으로 작업이 어느 시간대에 몰리는지 파악한다.
- 필요 지표:
  - 시간대별 등록 건수
  - 시간대별 총 평수
  - 라인별 시간대별 평수
  - 18시 이후 작업 비중
  - 월별/일별 피크 작업일
- 병목 후보 시간대는 별도 경고로 표시한다.

### 3.12 데이터 품질 분석 페이지
- 분석 목적:
  - 분석 신뢰도를 떨어뜨리는 누락, 중복, 이상값을 확인한다.
- 필요 지표:
  - 전체 행 수
  - PID 중복 건수
  - 현장 누락 건수 / 누락률
  - 거래처 누락 건수
  - 품명 누락 건수
  - 라인 누락 건수
  - 평수 0 또는 음수 건수
  - 가로/세로 0 또는 음수 건수
  - 대형 규격 이상치
- 데이터 품질 이슈는 대표/팀장이 공통으로 확인할 수 있어야 한다.

### 3.13 이상징후 자동 감지
- 규칙 기반 감지 후 AI가 설명한다.
- 1차 규칙:
  - 전월 대비 총 평수 30% 이상 증가 → 물량 급증
  - 특정 라인이 전체 평수의 60% 이상 처리 → 라인 편중
  - 상위 5개 거래처가 전체 평수의 50% 이상 → 고객 집중 리스크
  - 상위 5개 품목이 전체 평수의 50% 이상 → 품목 집중도 높음
  - 현장 누락률 5% 이상 → 데이터 품질 주의
  - 특정 시간대가 전체 작업의 20% 이상 → 시간대 병목 가능성
  - 특정 제품군이 전체 평수의 60% 이상 → 제품군 집중도 높음
- 감지 결과는 대표 대시보드와 AI 리포트에 반영한다.

### 3.14 AI 챗봇 기능
- 챗봇은 엑셀 전체를 직접 AI에 넣지 않는다.
- 처리 흐름:
  - 사용자 질문
  - 질문 의도 해석
  - 필요한 SQL 또는 집계 로직 결정
  - DB에서 숫자 계산
  - AI가 자연어로 설명
- 답변 가능한 질문 예시:
  - 이번 달 포장 실적 요약해줘
  - 1-LINE과 2-LINE 차이 분석해줘
  - 상위 거래처 의존도 알려줘
  - 로이/코팅 제품 비중이 높은 거래처 찾아줘
  - 특정 월에 물량이 급증한 이유를 추정해줘
  - 현장 누락이 많은 거래처 알려줘
  - 평균 평수가 큰 품목 TOP 10 보여줘
  - 오후 시간대에 작업이 몰리는지 분석해줘
  - 2-LINE에 특수 제품이 몰리는지 확인해줘
  - 특정 거래처가 주로 주문한 제품군 알려줘
- 모든 답변은 근거 숫자와 함께 제시한다.

### 3.15 AI 리포트 자동 생성
- 대시보드 상단 또는 별도 페이지에 AI 리포트 영역 제공
- 리포트 구조:
  - 전체 요약
  - 이번 기간 핵심 변화
  - 라인별 특징
  - 품목/제품군 특징
  - 거래처/현장 특징
  - 데이터 품질 이슈
  - 관리자가 확인해야 할 액션 아이템
- 리포트는 `가능한 해석`과 `추가 데이터가 필요한 해석`을 구분한다.

### 3.16 대표용 고도화 결과물 방향
- 기존 대시보드처럼 숫자만 보여주는 것이 아니라 아래 구조의 브리핑을 제공한다.
  - 오늘/이번 업로드 핵심 브리핑
  - 가장 먼저 볼 리스크
  - 변화 원인 후보
  - 집중도/편중 경고
  - 데이터 품질 경고
  - 바로 취할 수 있는 액션 제안
- 대표는 `조회`보다 `판단`에 더 가까운 결과물을 받아야 한다.

## 4. 비기능 요구사항

### 4.1 성능
- 대용량 파일도 원본 전체를 AI에 직접 전달하지 않는다.
- 저장된 행 데이터에서 필요한 데이터만 검색 후 AI에 전달한다.
- 대표/팀장 화면은 전체 데이터를 한 번에 그리지 않고, 집계/상위 목록/필터를 중심으로 구성한다.

### 4.2 유지보수성
- 업로드, 구조화, 분석, AI, 대시보드, 데이터 품질 기능을 모듈 단위로 분리한다.
- 업로드 구조 변경 시 대표 대시보드, 데이터 품질, AI 분석이 같은 구조화 데이터와 스냅샷을 재사용할 수 있어야 한다.

### 4.3 보안
- Gemini/OpenAI 계열 API 키는 서버 환경변수로만 관리한다.
- 업로드 파일 접근은 API 라우트를 통해 제어한다.

## 6. UX 방향
- 사용자는 엑셀을 업로드하면 바로 대시보드를 볼 수 있어야 한다.
- 숫자는 카드로 직관적으로 보여준다.
- 차트는 지나치게 복잡하지 않게 한다.
- AI 인사이트는 각 페이지 상단 또는 우측 패널에 표시한다.
- 대표 화면은 `무엇이 문제인지`, `왜 그런지`, `어떻게 개선할지`가 드러나야 한다.
- 팀장 화면은 운영/예외/확인 항목 중심이어야 한다.
